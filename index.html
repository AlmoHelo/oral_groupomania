Les demandes de l'entreprise

  • Présentation des fonctionnalités simple
  • Création d'un compte simple et possible depuis un mobile
  • Profil avec peu d'informations
  • Suppression du compte possible
  • Contenus multimédias et textuels
  • Voir les dernières participations des employés
  • Le chargé de communication doit pouvoir modérer les interactions

Présentation de la stack utilisée

  • Vuejs : c'est un framework utilisé pour le Front-end
  • NodeJs et express : c'est une plateforme logicielle en javascript et son infrastructure
  • Mysql : c'est un service de base de données

Le front-end

  • Dossier assets : il contient nos images utilisées sur le site ainsi que les différentes polices.
  • Dossier components : il contient tous nos composants. Un composant nous permet de regrouper un ensemble d'éléments HTML pour le réutiliser et pour faciliter sa maintenance.
  • Dossier router : il contient les routes de nos différentes pages.
  • Dossier service : il contient des fonctions utilisées à divers endroits du site.
  • Dossier views : il contient toutes les pages du site.
  • Fichier main.js : il contient les options de configuration.

Le back-end

  • Dossier controllers : Il contient la logique métier de nos routes.
  • Dossier images : Il contient les images enregistrées par les utilisateurs du site.
  • Dossier middleware : Ce sont des logiciels qui fournissent aux applications des fonctionnalités et des services communs. Ici :
    • auth.js nous permet de protéger les routes sélectionnées et de vérifier que l'utilisateur est authentifié
    • multer.js nous permet de gérer les fichiers entrants, les images, dans les requêtes HTTP
  • Dossier routes : Il contient la logique de nos routes.
  • Fichier .gitignore : Il contient les fichiers à ignorer. Ici node_modules et aussi .env.
  • Fichier app.js : C'est le fichier de démarrage du projet. Il contient tout le projet.
  • Fichier serve.js : Il contient notre serveur Node. Node est un runtime qui nous sert pour écrire toutes nos tâches côté serveur en Javascript.

La base de donnée

  • Table user : contient toutes les coordonnées de notre utilisateur.
  • Table item : elle contient les articles postés par les utilisateurs.
  • Table comment : elle contient les commentaires de chaque utilisateur selon l'article commenté.
  • Table userLikes : elle répertorie les likes de chaques utilisateurs selon l'article aimé.
  • Table userDislikes : elle répertorie cette fois les dislikes de chaques utilisateurs selon l'article qui n'est pas aimé.
  • Table report : unique accessible par l'administrateur désigné par isAdmin dans la table user. Elle référence tous les articles et commentaires qui ont été signalé.

Chaque table contient une clé primaire qui permet d'identifier de manière unique un enregistrement sur la table. Ainsi que des clés étrangères qui sont des contraintes pour garantir l'intégrité référentielle entre deux tables. Une clé étrangère identifie une colonne ou un ensemble de colonnes d'une table comme référençant une colonne ou un ensemble de colonnes d'une autre table.
